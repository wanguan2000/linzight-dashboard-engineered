# RWS EDC Demo 自动开发总控计划

目标：基于现有前端界面、数据显示和业务逻辑，继续开发一个可试跑的 RWS EDC Demo 系统，使用 50 个模拟患者数据完成端到端验证,及PI研究者，CRC，系统管理员角色

## 技术栈
- 前端：现有前端为基础，优先工程化改造
- 后端：Python FastAPI
- 数据库：PostgreSQL，开发阶段可先兼容 SQLite
- 文件存储：本地 uploads 目录，后续可替换 MinIO/S3
- 数据：生成 50 个模拟患者，覆盖 NPSLE、Non-NPSLE、MS、NMOSD、HC
- 权限：sys_admin、project_admin、investigator、crc、data_manager、viewer

## 必须连续完成的阶段
1. 前端工程化改造
2. 设计数据库表结构
3. 定义 API 接口
4. 建立前后端联调的数据协议
5. 后端开发
6. 权限与账户体系
7. 文件上传与隐私处理
8. 数据导入导出
9. 数据完整性与校验
10. 审计日志
11. 测试

## 自动推进规则
每完成一个阶段，必须：
1. 运行相关测试或构建命令
2. 修复发现的问题
3. 更新 docs/PROGRESS.md
4. 执行 git add . && git commit
5. 自动进入下一阶段

除非遇到以下情况，否则不要停下来问用户：
- 缺少必须的账号、密钥或外部服务
- 当前仓库完全缺失必要文件，无法判断技术栈
- 连续 3 次修复同一个错误仍失败
- 涉及删除大量文件或重构不可逆操作

## v1 不做
- 医院 HIS/EMR 真实对接
- GCP 完整合规流程
- 复杂 query management
- 电子签名 CA 认证
- 入排标准自动判断
- 多中心复杂流程

登录 → 患者列表 → 患者详情 → CRF 录入 → 样本登记 → 多组学检测登记 → 文件上传 → Patient Journey → 数据分析 → 导出
